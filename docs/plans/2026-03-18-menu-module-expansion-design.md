# 主菜单扩展设计

## 目标
- 将主菜单中原本的 `todo` 按钮替换为独立模块入口
- 保持现有 `admin.py -> group.py -> services/*` 的架构，不重做机器人主流程
- 优先落地无需长期后台进程即可运行的功能

## 设计
- 配置层统一放入 `group config` 的嵌套字典中，避免再新增大量离散 KV key。
- `config_store.get_group_config()` 改为递归 merge，保证嵌套模块配置不会因单字段保存被整体覆盖。
- 新增 `admin_extra.py` 负责图片对应的新模块菜单和输入状态，避免继续膨胀 `admin.py`。
- 新增 `extra_features.py` 负责群内运行时功能，包括：
  - 广告封杀
  - 命令关闭
  - 积分与活跃度
  - 加密货币与 U 价查询
  - 邀请链接、定时消息、抽奖
  - 语言白名单、昵称变更检测、关联频道、基础色情处理

## 运行策略
- 定时消息不依赖常驻任务，改为“群内有新 update 时顺带检查并触发到期消息”，兼容当前 webhook 模式。
- 邀请排行、积分排行、活跃排行都通过单独的“用户索引 key”维护，兼容内存 KV 与 Upstash KV。
- U 价查询目前直接使用公开接口：
  - Binance P2P
  - OKX C2C
  - HTX 若无可用 CNY OTC 数据则明确返回不可用

## 当前保留项
- 钱包地址资产查询仍需接入专用链上资产 API
- 五子棋仅保留入口
- 色情处理当前仅做文本/文件名/贴纸名的启发式检测
- 管理权限菜单已可配置，但普通管理员接管私聊配置入口尚未放开
